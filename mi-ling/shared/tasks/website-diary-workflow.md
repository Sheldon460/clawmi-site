# Clawmi 网站日记更新任务分配 (V1.0)

## 任务分工（三级协作）

### 1️⃣ 小幂 - 内容创作
**职责**：
- 专注日记的内容创作和撰写
- 每日 12:00 前完成中午日记
- 每日 18:00 前完成晚间日记
- 将日记内容提交至 `/Users/sheldon/clawd/mi-army/xiao-mi/diary/`

**输出物**：
- `YYYY-MM-DD-noon.md`（中午日记）
- `YYYY-MM-DD-evening.md`（晚间日记）

---

### 2️⃣ 幂领 - 统筹审核
**职责**：
- 统筹整个日记更新流程
- 审核小幂提交的日记内容
- 在部署失败时启动替补方案
- 协调小幂和幂码之间的协作

**审核要点**：
- 内容质量检查
- 格式规范检查
- 时效性确认

---

### 3️⃣ 幂码-编程 - 技术部署
**职责**：
- 负责技术部署执行
- 每日 12:00 和 18:00 后检查日记更新
- 执行部署流程：
  1. 更新 `data/diary.json`
  2. Git commit 提交更改
  3. Git push 推送到远程
  4. 验证 Vercel 自动部署
  5. 确认网站可访问

**部署脚本**：
- 主脚本：`/Users/sheldon/.clawmi/scripts/diary-reminder.sh`
- 日志目录：`/Users/sheldon/.clawmi/logs/`

---

## 协作流程

```
小幂创作 → 幂领审核 → 幂码部署 → 完成通知
   ↑                                    ↓
   └──────── 问题反馈 ← 失败处理 ────────┘
```

## 定时任务配置

| 时间 | 任务 | 执行者 |
|------|------|--------|
| 每天 12:00 | 中午日记更新 | 幂码 |
| 每天 18:00 | 晚间日记更新 | 幂码 |

## 应急处理

**当幂码部署失败时**：
1. 幂领立即介入
2. 检查日志定位问题
3. 手动执行 Git 推送
4. 验证部署状态
