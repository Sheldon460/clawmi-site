### 1.2 核心瓶颈识别

#### 🔴 瓶颈 1：素材抓取（30 分钟）

**问题**:
- 单篇文章抓取平均耗时 2.7 分钟
- 11 篇文章串行执行，无并行化
- 部分网站反爬机制导致重试

**根因**:
```
当前模式: 串行抓取
URL 1 → 抓取 → 保存 → URL 2 → 抓取 → 保存 ...
```

**优化方案**:
```
目标模式: 并行抓取
URL 1 ─┐
URL 2 ─┼→ 并行抓取 → 批量保存
URL 3 ─┘
预计提升: 30 分钟 → 10 分钟 (3x 加速)
```

#### 🔴 瓶颈 2：文章创作（45 分钟）

**问题**:
- humanize-zh 技能未安装，需手动 L2 处理
- 创作 + 润色串行执行
- 风格对齐需要多次迭代

**根因**:
- 技能依赖缺失
- 人工介入打断自动化流程

**优化方案**:
1. 安装 humanize-zh 技能
2. 创建一键创作脚本
3. 预置风格模板减少迭代

**预计提升**: 45 分钟 → 25 分钟 (1.8x 加速)

#### 🔴 瓶颈 3：API 配置（10 分钟）

**问题**:
- IP 白名单配置被遗漏
- 环境变量手动设置
- 缺乏配置验证机制

**根因**:
- 配置步骤未文档化
- 缺乏预检脚本

**优化方案**:
1. 创建配置检查脚本
2. 预置配置模板
3. 首次运行时自动检测

**预计提升**: 10 分钟 → 2 分钟 (5x 加速)

### 2.2 配图质量优化路径

#### 当前问题

| 问题 | 影响 | 频率 |
|------|------|------|
| 提示词不够精准 | 配图与内容匹配度低 | 40% |
| 风格不统一 | 视觉体验割裂 | 30% |
| 封面吸引力不足 | 点击率受影响 | 50% |

#### 优化策略

**1. 提示词模板化**

```yaml
# 封面图提示词模板
cover_template: |
  主题: {article_topic}
  风格: 极简主义，高级感，留白
  色调: 暖色系，柔和光线
  元素: {key_element}，抽象化表达
  比例: 16:9
  质量: 4K，专业摄影级

# 内文图提示词模板
inline_template: |
  场景: {scene_description}
  风格: 与封面保持一致
  色调: {color_palette}
  元素: 简洁，聚焦主题
  比例: 4:3
```

**2. 风格统一机制**

```
建立视觉风格库:
├── 科技极客风/        # 冷色调，几何元素
├── 温暖人文风/        # 暖色调，自然元素
├── 商务专业风/        # 中性色，简洁线条
└── 创意艺术风/        # 高饱和，抽象元素

每篇文章选定一种风格，全部配图保持一致
```

**3. 封面吸引力优化**

```
封面检查清单:
- [ ] 主题是否一眼可识别？
- [ ] 是否有视觉焦点？
- [ ] 色调是否吸引眼球？
- [ ] 是否适合小尺寸展示？
- [ ] 是否与标题呼应？
```

#### 坑 1：浏览器发布方式已弃用

```markdown
## 🚨 重大变更：浏览器发布方式已弃用

**日期**: 2026-03-10
**原因**: 富文本编辑器粘贴不稳定
**替代方案**: API 发布方式

### 弃用原因详解
1. 公众号编辑器使用 TinyMCE 类富文本编辑器
2. CDP 键盘事件无法触发编辑器的自定义粘贴处理
3. 焦点竞争问题（多 Chrome 实例）
4. 成功率仅 50%，不可接受

### 迁移指南
```bash
# 旧方式（已弃用）
npx -y bun scripts/wechat-browser.ts

# 新方式（推荐）
npx -y bun scripts/wechat-api.ts <html> --title "标题"
```

### 遗留代码处理
- 保留浏览器方式作为备用
- 默认使用 API 方式
- 文档中标注「备用」
```
