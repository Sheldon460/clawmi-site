## 🔍 问题诊断与解决

### 问题 1：浏览器发布粘贴失败

**现象**:
```
Body content verification failed: editor appears empty after paste.
Placeholder not found: MDTOHTMLIMGPH_1
```

**根因分析**:
1. 公众号编辑器使用富文本编辑器（TinyMCE 类）
2. CDP 键盘事件无法触发编辑器的自定义粘贴处理
3. 焦点竞争：33 个 Chrome 实例运行

**解决方案**:
- 使用 API 发布方式（wechat-api.ts）
- 完全绕过浏览器粘贴
- 稳定可靠，30 秒完成

**经验沉淀**:
> 公众号发布优先使用 API 方式，浏览器方式仅作为备用。

---

### 问题 2：API 凭证配置

**现象**:
```
errcode: 40164
errmsg: invalid ip 111.60.29.160, not in whitelist
```

**根因分析**:
- IP 白名单未配置
- 微信公众号安全机制

**解决方案**:
- 在公众号后台添加服务器 IP
- 配置 `~/.canghe-skills/.env`

**经验沉淀**:
> API 发布前必须配置 IP 白名单，这是最常见的新手错误。

### 核心知识 1：API 发布最佳实践

```bash
# 1. 配置凭证
cat > ~/.canghe-skills/.env << 'EOF'
WECHAT_APP_ID=xxx
WECHAT_APP_SECRET=xxx
default_publish_method: api
EOF

# 2. 配置 IP 白名单（关键！）
# 公众号后台 → 开发 → 基本配置 → IP 白名单

# 3. 发布命令
cd ~/.openclaw/skills/canghe-post-to-wechat
npx -y bun scripts/wechat-api.ts <html> --title "标题"
```
