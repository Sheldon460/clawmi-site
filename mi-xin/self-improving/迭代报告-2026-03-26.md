## 遇到的问题汇总

### 问题 1：标题层级混乱

**现象**：
- 文章出现双标题（H1 + YAML frontmatter）
- H2 标题有 "01/02/03" 前缀，与 wechat-markdown-pro 自动序号冲突
- 大量 H3 标题（###）与 H2 序号层级冲突

**原因**：
- mi-wen 创作时未遵循 SOP V3.0 标题层级规范
- 任务指令中未明确强调标题规范

**解决方案**：
1. 删除 H1 标题
2. H2 标题删除 "01/02/03" 前缀
3. H3 标题改为 `**加粗**`

**预防措施**（已更新 SOP）：
- 阶段 4：在 mi-wen 任务指令中强制要求标题层级规范
- 阶段 6：添加排版预检环节，自动检查 H1/H3

### 问题 2：配图复用旧文章图片

**现象**：
- 使用了之前文章的图片（01_body.png, 02_native.png, 00_cover.png）
- 图片内容与本文主题不匹配

**原因**：
- mi-hua 配图任务超时，未生成新图
- 未检查配图是否为本文专用
- SOP 中缺少配图内容匹配检查项

**解决方案**：
1. 重新生成专用配图（2026-03-26-龙虾-XX.png）
2. 更新文章图片路径
3. 使用日期前缀命名避免混淆

**预防措施**（已更新 SOP）：
- 阶段 5：添加"配图内容与文章主题匹配"检查项
- 阶段 5：要求文件名为新命名（含日期前缀）

### 问题 3：sed 命令误用导致格式混乱

**现象**：
- 使用 `sed -i '' 's/^### /\*\*/g; s/$/\*\*/g'` 修复 H3 标题
- 结果每行末尾都添加了 `**`，导致全文格式混乱

**原因**：
- sed 命令语法错误，`s/$/\*\*/g` 会在每行末尾添加 `**`
- 未验证修复结果直接发布

**解决方案**：
1. 使用 Python 脚本精确替换
2. 从原始文件重新修复

**预防措施**：
- 禁止使用复杂 sed 命令修改 Markdown
- 使用 Python 脚本进行精确文本替换
- 修复后必须人工验证

---

### 问题 4：个人案例过度突出

**现象**：
- 文章大量引用杨天润个人故事（72小时冲进贡献者榜前30）
- 读起来像个人传记，而非通用方法论

**原因**：
- mi-wen 创作时未弱化个人案例
- 未将个人经验转化为通用框架

**解决方案**：
1. 删除具体个人背景描述
2. 将"杨天润说"改为"有人总结/有人提出"
3. 保留核心观点，改为通用表述

**预防措施**（已更新 SOP）：
- 阶段 4：在 mi-wen 任务指令中添加"弱化个人案例"要求
- 强调将个人经验转化为通用方法论
