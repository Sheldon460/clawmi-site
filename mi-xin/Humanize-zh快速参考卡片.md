# 📝 Humanize-zh 快速参考卡片（打印版）

## 一句话定义
将AI生成的机械化文本转换为自然、有人情味的人类写作风格。

---

## 📊 三级处理速查表

| 维度 | L1 轻度（商业/学术） | L2 中度（公众号深度） | L3 重度（小红书/短视频） |
|------|-------------------|-------------------|---------------------|
| **触发词** | 白皮书、报告、ROI、数据分析 | 公众号、深度、长文、观点 | 小红书、抖音、脚本、种草 |
| **Humanize强度** | 20-30% | 50-60% | 80-90% |
| **句式** | 70%长+30%短 | 50%长+50%短 | 10%长+90%短 |
| **口语化密度** | 极低（1000字0-1个） | 中（1000字3个） | 极高（每3句1个） |
| **网络用语** | ❌ 禁用 | ⚠️ 少量（"绝了"） | ✅ 密集（"救命""家人们"） |
| **个人经历** | ❌ 不需要 | ✅ 必须（1个） | ✅ 密集（每段都有"我"） |
| **emoji** | ❌ 禁用 | ⚠️ 结尾可用 | ✅ 大量使用（🔥💯😭） |
| **情绪词** | ❌ 客观克制 | ⚠️ 适度（"挺意外的"） | ✅ 强烈（"我真的会谢"） |

---

## 🛠️ 7大技巧速查（按级别应用）

| 技巧 | L1 | L2 | L3 | 示例 |
|------|----|----|----|------|
| **1. 打破完美结构** | ❌ | ✅ | ✅ | L2: "另外值得一提的是" → L3: 直接堆感受 |
| **2. 加入不完美元素** | ⚠️ | ✅ | ✅ | L1: 0个 → L2: "说白了" → L3: "救命""绝了" |
| **3. 用具体代替抽象** | ✅ | ✅ | ✅ | L1: "数据显示" → L2: "我之前测过" → L3: "我亲测" |
| **4. 加入个人色彩** | ❌ | ✅ | ✅ | L2: "说实话" → L3: "我真的""我直接" |
| **5. 长短句混搭** | ✅ | ✅ | ✅ | L1: 70%长 → L2: 50%长 → L3: 10%长 |
| **6. 用主动语态** | ✅ | ✅ | ✅ | 通用："我们决定" > "决定被做出" |
| **7. 加入停顿和思考** | ❌ | ⚠️ | ✅ | L3: "嗯...""等一下""让我想想" |

## ✅ 三级自检清单

### L1 轻度检查清单（商业/学术）
- [ ] 是否保留了所有数据和术语？
- [ ] 是否使用了主动语态？
- [ ] 是否实现了长短句混搭（70%长+30%短）？
- [ ] 是否去除了"综上所述"等AI味词汇？（可选）
- [ ] 读一遍，是否像专业人士写的？

**通过标准**：5项全部满足 → 可发布

---

### L2 中度检查清单（公众号深度）
- [ ] 是否加入了至少1个个人经历/案例？
- [ ] 是否有至少3个口语化表达（每1000字）？
- [ ] 是否避免了全篇"首先/其次/最后"？（允许少量）
- [ ] 是否加入了个人判断/情感升华？
- [ ] 核心论点是否仍然清晰？
- [ ] 读一遍，会不会觉得"这个人挺真实的"？
- [ ] 会不会觉得"太随意了，不专业"？（应避免）

**通过标准**：前6项满足 + 第7项不满足 → 可发布

---

### L3 重度检查清单（小红书/短视频）
- [ ] 是否每3句话就有1个网络用语？
- [ ] 是否有至少3个第一人称"我"（每100字）？
- [ ] 是否避免了超过100字的长段落？
- [ ] 是否有明确的"钩子"（前3句抓住注意力）？
- [ ] 是否有明确的行动号召（结尾引导互动）？
- [ ] 是否使用了emoji（🔥💯😭🆘）？
- [ ] 读一遍，像不像闺蜜/兄弟在跟你聊天？
- [ ] 有没有想立刻转发/评论的冲动？

**通过标准**：前6项满足 + 第7/8项至少满足1项 → 可发布

## 🚀 三、集成到SOP的代码示例

### 在 SOP_GZH.md 中的集成方式

```markdown
## 阶段 4：初稿打磨（Humanize-zh 分级处理版）

- 触发：用户确认某一大纲。
- 动作：
  1. 读取 USER.md 中的写作风格
  2. 调用公众号爆文工作流.skill生成初稿
  3. **自动判定内容级别（关键词匹配）：**
     ```
     如果指令包含"白皮书/报告/提案/ROI/数据分析" → 判定为 L1 轻度
     如果指令包含"公众号/深度/长文/观点/思考" → 判定为 L2 中度
     如果指令包含"小红书/抖音/短视频/脚本/种草" → 判定为 L3 重度
     如果同时匹配多个 → 默认取低级别（保守策略）
     ```
  4. **调用 humanize-zh skill 进行对应级别的处理：**
     - L1：仅应用技巧①+②（主动语态+长短句混搭）
     - L2：应用技巧①-⑤（打破结构+口语化+具体化+个人色彩+长短句）
     - L3：应用全部7个技巧（最大化人味）
  5. **执行对应级别的自检清单：**
     - L1：检查5项，全部满足才通过
     - L2：检查7项，前6满足+第7不满足才通过
     - L3：检查8项，前6满足+第7/8至少1项满足才通过
  6. 存入「mi-xin/03-内容工厂/2-初稿打磨区/」

- 异常处理：
  - 如果自检不通过 → 返回步骤3，调整 Humanize 强度或重写
  - 如果用户明确要求"更专业"或"更口语" → 手动调整级别（L+1 或 L-1）
```

## 📊 四、效果验证方法

### A/B 测试方案

**测试目标**：验证 Humanize-zh 分级处理对阅读/互动数据的影响

**测试设计**：

| 组 | 处理方式 | 发布平台 | 样本量 | 观测指标 |
|---|---------|---------|--------|---------|
| A（对照组） | 不使用 Humanize，直接发布AI初稿 | 相同平台 | 10篇 | 阅读完成率、点赞、评论、转发 |
| B（实验组1） | 使用 L1 轻度 Humanize | 相同平台 | 10篇 | 同上 |
| C（实验组2） | 使用 L2 中度 Humanize | 相同平台 | 10篇 | 同上 |
| D（实验组3） | 使用 L3 重度 Humanize | 相同平台 | 10篇 | 同上 |

**数据分析维度**：
1. **总体效果**：Humanize vs 不Humanize，哪个互动率更高？
2. **分级效果**：L1/L2/L3，哪个级别在特定平台效果最好？
3. **内容类型匹配度**：深度长文用L2 vs L1，哪个效果更好？

**结论应用**：
- 如果 L2 > L1 → 更新 SOP，将公众号深度长文默认设为 L2
- 如果 L3 在小红书效果最好 → 固化 L3 为小红书标准流程
- 如果 Humanize 整体效果 < 不Humanize → 重新审视 skill 应用策略
