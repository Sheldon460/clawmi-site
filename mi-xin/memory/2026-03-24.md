## 问题发现与修复

**用户反馈**：md2wechat 排版更好看，但图片显示有问题

**问题分析**：
1. md2wechat 的 `sync-md` 命令不支持自动上传正文中的本地图片
2. 只会处理封面图（--cover 参数）
3. Markdown 中的 `![alt](path)` 语法被直接转为文本而非图片
4. YAML frontmatter 被错误渲染为标题

**修复尝试**：
- 使用 `sync-md` 重新发布（修复版）
- Media ID: `_9ibZdusdmeqZ4ujHJe6rqSwoCjQHIRJSRmOGbv5oaz-uLcMZOZD2KzSZ23opvqb`
- 结果：封面图正常，正文图片仍无法显示

**根本原因**：
md2wechat 设计初衷是配合在线图床使用，不支持自动上传本地图片到微信素材库。

**解决方案**：
如需使用 md2wechat 并显示正文图片，需要：
1. 先用 `md2wechat upload_image` 上传每张图片获取 URL
2. 在 Markdown 中使用微信素材库 URL
3. 或使用在线图床（如腾讯云COS、阿里云OSS）

**对比结论更新**：
- **canghe-markdown-to-html**：✅ 支持自动上传本地图片，更适合我们的 workflow
- **md2wechat**：⚠️ 需要额外的图片处理步骤，但排版更美观

## 新技能开发：wechat-markdown-pro

**创建时间**：2026-03-24 12:45
**目标**：融合 canghe-markdown-to-html 和 md2wechat 两者的优势

### 核心功能
1. **自动图片上传**：像 canghe 一样自动上传本地图片到微信素材库
2. **精美排版**：像 md2wechat 一样使用橙色主题 + 序号标题
3. **YAML支持**：正确处理 frontmatter，不渲染为文本
4. **一体化命令**：convert + publish 两个命令完成全流程

### 技术实现
- **Runtime**: Bun + TypeScript
- **转换引擎**: 自定义 Markdown → HTML（基于 md2wechat 样式）
- **图片上传**: 复用 canghe 的微信 API 调用逻辑（add_material 接口）
- **命令设计**:
  - `wechat-markdown-pro convert <input.md>` - 转换并预览
  - `wechat-markdown-pro publish <input.md> --cover <path>` - 转换并发布
