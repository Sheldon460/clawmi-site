# Humanize-zh 分级应用处理方案 V1.0

## 🎯 方案目标
建立可执行的内容分级 Humanize 流程，确保不同平台/类型的内容获得恰到好处的"人味"。

---

## 📊 一、内容分级判断矩阵

### 自动判断逻辑（关键词匹配）

```
用户指令 → 关键词匹配 → 级别判定
```

| 触发关键词 | 内容类型 | 判定级别 | Humanize强度 |
|-----------|---------|---------|-------------|
| "白皮书"、"报告"、"提案"、"分析"、"ROI"、"数据" | 商业/学术 | **L1 轻度** | 20-30% |
| "公众号"、"深度"、"长文"、"观点"、"思考" | 深度内容 | **L2 中度** | 50-60% |
| "小红书"、"抖音"、"短视频"、"脚本"、"朋友圈"、"种草" | 短平快内容 | **L3 重度** | 80-90% |

### 模糊场景处理
如果同时匹配多个级别（如"小红书深度种草文案"），**默认取低级别**（L2而非L3），确保专业度不被过度稀释。

---

## ⚙️ 二、分级处理执行流程

### **L1 轻度处理流程（商业/学术）**

**适用场景**：白皮书、商业提案、数据分析报告、学术论文

**执行步骤**：

```
Step 1: 初稿生成
├─ 使用标准AI生成专业内容
└─ 保留所有数据、图表、专业术语

Step 2: L1 Humanize（轻度）
├─ 仅应用以下2个技巧：
│  ├─ ① 用主动语态替换被动语态
│  │     例："决定被做出" → "我们决定"
│  └─ ② 长短句混搭（保持70%长句+30%短句）
│        例：在关键观点前使用短句强调
└─ 严格保留：数据、术语、逻辑连接词（"首先/其次"）

Step 3: 质量检查
├─ [ ] 所有数据是否完整保留？
├─ [ ] 专业术语是否准确？
├─ [ ] 是否还有"综上所述"类词汇？（允许少量）
└─ [ ] 读一遍，是否像专业人士写的？
```

**效果目标**：去除机械感，但保持专业严谨。读者感觉"这是专家写的，但不难读"。

---

### **L2 中度处理流程（公众号深度长文）**

**适用场景**：公众号深度文章、知识付费内容、观点输出类长文

**执行步骤**：

```
Step 1: 初稿生成
├─ 使用标准AI生成内容框架
└─ 确保有明确论点、论据、结论

Step 2: L2 Humanize（中度）
├─ 应用以下5个技巧：
│  ├─ ① 打破完美结构
│  │     例：用"另外值得一提的是"代替"第二"
│  │     例：用"还有个有意思的事"代替"第三"
│  ├─ ② 加入不完美元素（每1000字2-3个）
│  │     例："说白了..."、"老实说..."、"你懂的"
│  ├─ ③ 用具体代替抽象
│  │     例："我上次用的时候..."代替"用户体验良好"
│  ├─ ④ 加入个人色彩
│  │     例：开头或结尾加一句个人判断
│  │     例："说实话，看到这个数据我挺意外的"
│  └─ ⑤ 长短句混搭（50%长句+50%短句）
│        例：关键观点用短句，背景用长句
└─ 适度保留：核心论点、逻辑框架

Step 3: 内容增强
├─ 在正文插入1-2个个人经历/案例（占20-30%）
├─ 在开头或结尾加入个人判断/情感升华
└─ 确保每1000字有至少3个口语化表达

Step 4: 质量检查
├─ [ ] 是否有至少1个个人经历/案例？
├─ [ ] 是否有至少3个口语化表达？
├─ [ ] 是否避免了全篇都是"首先/其次/最后"？
├─ [ ] 核心论点是否仍然清晰？
├─ [ ] 读一遍，会不会觉得"这个人挺真实的"？
└─ [ ] 会不会觉得"太随意了，不专业"？（避免过度）
```

**效果目标**：专业中带温度，观点中带人情味。读者感觉"这个人懂行，还挺好聊的"。

---

### **L3 重度处理流程（小红书/短视频脚本）**

**适用场景**：小红书笔记、抖音短视频脚本、朋友圈文案、种草内容

**执行步骤**：

```
Step 1: 初稿生成
├─ 快速生成核心信息（产品特点、使用场景、效果）
└─ 确保有明确的"钩子"和"行动号召"

Step 2: L3 Humanize（重度）
├─ 应用全部7个技巧（最大化"人味"）：
│  ├─ ① 打破完美结构（完全不用"首先/其次"）
│  │     直接堆感受，想到哪说到哪
│  ├─ ② 加入不完美元素（每3句就有1个）
│  │     例："救命"、"绝了"、"真的"、"我裂开了"、"真的哭死"
│  ├─ ③ 用具体代替抽象（夸张但具体）
│  │     例："这个包能装下我的整个世界"
│  ├─ ④ 加入个人色彩（全是"我觉得"、"我发现"、"我的天"）
│  ├─ ⑤ 长短句混搭（短句为主90%，长句只用来堆情绪）
│  ├─ ⑥ 用主动语态（"我直接爱死"而不是"这个产品被我喜爱"）
│  └─ ⑦ 加入停顿和思考（"嗯..."、"等一下"、"让我想想"）
└─ 完全不保留：逻辑连接词、工整结构、专业术语

Step 3: 格式优化
├─ 每段不超过100字（小红书）/ 每句话独立成行（短视频）
├─ 大量使用emoji（🔥💯✨😭🆘）
├─ 关键信息用【】或「」突出
└─ 结尾加明确的行动号召（"评论区告诉我"、"蹲一个反馈"）

Step 4: 质量检查
├─ [ ] 是否每3句话就有1个网络用语？
├─ [ ] 是否有至少3个第一人称"我"？
├─ [ ] 是否避免了超过100字的长段落？
├─ [ ] 是否有明确的"钩子"（前3句抓住注意力）？
├─ [ ] 是否有明确的行动号召（结尾引导互动）？
├─ [ ] 读一遍，像不像闺蜜/兄弟在跟你聊天？
└─ [ ] 有没有想立刻转发/评论的冲动？
```

**效果目标**：最大化口语化和真实感，情绪价值拉满。读者感觉"这简直就是我在说话"，立刻想转发/评论。

## 🛠️ 四、技能调用示例代码

### 在 SOP 中集成 Humanize-zh

```markdown
## 阶段 4：初稿打磨（修订版）

- 触发：用户确认某一大纲。
- 动作：
  1. 读取 USER.md 中的写作风格
  2. 调用 `wechat-article-pro` skill 生成初稿
  3. **判断内容类型，调用 `humanize-zh` skill 进行分级处理：**
     - 如果内容含"白皮书/报告/提案/ROI" → L1 轻度处理
     - 如果内容含"公众号/深度/长文/观点" → L2 中度处理
     - 如果内容含"小红书/抖音/短视频/脚本" → L3 重度处理
  4. 存入「mi-xin/03-内容工厂/2-初稿打磨区/」
```
