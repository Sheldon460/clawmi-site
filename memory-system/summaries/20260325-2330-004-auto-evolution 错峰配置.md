## 🎯 讨论主题
为避免 26 个 agent 同时执行定时任务导致 API 调用频率过高，配置错峰执行策略。

## 💡 错峰策略

### 核心思路
将 26 个 agent 分散到不同时间执行，避免 API 调用峰值。

### 具体方案

#### 1. 每小时快速扫描 (0-5 分钟循环)
- **策略**: 26 个 agent 分散到每小时的 0-5 分钟
- **峰值**: 每分钟最多 5 个 agent
- **示例**:
  - 分钟 0: main, mi-ying, mi-bo, mi-fa, mi-dang
  - 分钟 1: mi-ling, mi-sheng, mi-hu, mi-ren, xiao-mi
  - ...

#### 2. 每日深度复盘 (23:00-23:40)
- **策略**: 分 5 组，每组间隔 10 分钟
- **峰值**: 每组最多 6 个 agent
- **分组**:
  - 23:00 - Group 1: 6 个 agent
  - 23:10 - Group 2: 6 个 agent
  - 23:20 - Group 3: 6 个 agent
  - 23:30 - Group 4: 6 个 agent
  - 23:40 - Group 5: 2 个 agent

#### 3. 每周分析 (周日 23:00-23:40)
- 同样分 5 组，错峰执行

#### 4. 月度报告 (每月 1 日 23:00-23:40)
- 同样分 5 组，错峰执行
