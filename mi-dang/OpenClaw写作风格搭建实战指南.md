## 前言：从「会写」到「能沉淀」

很多创作者在使用 AI 写作时遇到的核心问题不是「写不出来」，而是**「写出来不像我」「每次都要重新教」「风格不稳定」**。

本指南基于 OpenClaw 的长期记忆与 Skill 系统，帮助你搭建一套**可复用、可迭代、可传承**的写作风格工作流。

---

## 第一部分：理解 OpenClaw 的记忆架构

### 1.1 三层记忆体系

```
┌─────────────────────────────────────────────────┐
│  Layer 3: Skill 层（最稳定）                     │
│  • 写作 SOP、固定模板、约束规则                  │
│  • 沉淀为 SKILL.md，长期复用                     │
├─────────────────────────────────────────────────┤
│  Layer 2: MEMORY.md（中长期）                    │
│  • 核心风格偏好、已验证方法                      │
│  • 跨会话持续生效                                │
├─────────────────────────────────────────────────┤
│  Layer 1: Daily Memory（短期）                   │
│  • 当日对话、试错记录                            │
│  • 定期筛选提炼到上层                            │
└─────────────────────────────────────────────────┘
```

### 1.2 记忆流动原则

```
Daily Memory  ──提炼──>  MEMORY.md  ──固化──>  Skill
   (短期)                  (中期)                 (长期)
    ↑                      ↑                      ↓
    └────── 迭代优化 ◄─────┴──── 持续复用 ◄──────┘
```

## 第二部分：搭建写作风格的四步流程

### Step 1: 风格拆解（输入 → 分析）

#### 1.1 收集样本

**需要准备的材料：**

| 类型 | 数量 | 选择标准 |
|------|------|----------|
| 自己的文章 | 5-10 篇 | 数据好、认可度高、风格明显 |
| 竞品/头部文章 | 3-5 篇 | 同赛道、长期稳定输出 |

#### 1.2 拆解维度

**对每篇文章分析以下维度：**

```
文章分析模板
============

【基本信息】
- 标题: 
- 发布时间: 
- 阅读/点赞/在看数据: 

【标题分析】
- 结构: [悬念式/数字式/对比式/痛点式]
- 关键词: 
- 情绪触发: 

【开头分析】
- 字数: 
- 钩子类型: [痛点共鸣/颠覆认知/利益承诺/场景代入]
- 结构: [SCQA/PREP/故事开场]

【正文结构】
- 段落数: 
- 平均每段字数: 
- 逻辑推进: [并列式/递进式/对比式]
- 案例使用: [个人经历/用户故事/数据/热点]

【语言风格】
- 人称: [你/我/我们]
- 句式特点: [长短句比例/口语化程度/修辞使用]
- 情绪节奏: [平铺直叙/起伏跌宕/层层递进]

【结尾分析】
- 字数: 
- 结尾类型: [行动召唤/情感共鸣/金句收尾/开放讨论]
- CTA: [引导在看/引导评论/引导转发]
```

#### 1.3 补充背景信息

**提供三类关键背景：**

```
┌────────────────────────────────────────┐
│ 1. 创作者画像                          │
├────────────────────────────────────────┤
│ - 身份定位（专家/博主/创业者）        │
│ - 内容领域（行业/垂类）                │
│ - 个人特色（经历/观点/风格）          │
└────────────────────────────────────────┘

┌────────────────────────────────────────┐
│ 2. 读者画像                            │
├────────────────────────────────────────┤
│ - 人群特征（年龄/职业/痛点）          │
│ - 阅读场景（通勤/睡前/碎片时间）      │
│ - 内容偏好（干货/故事/观点）          │
└────────────────────────────────────────┘

┌────────────────────────────────────────┐
│ 3. 数据反馈                            │
├────────────────────────────────────────┤
│ - 哪类文章打开率高                     │
│ - 哪类文章完读率高                     │
│ - 哪类文章转发率高                     │
└────────────────────────────────────────┘
```

### Step 2: 规则沉淀（分析 → 记忆）

#### 2.1 生成风格规则

**基于拆解结果，提炼成可执行规则：**

```
风格规则模板
==============

【标题规则】
- 首选结构: [具体结构]
- 字数控制: [X-Y字]
- 必备元素: [数字/冲突/悬念/情绪词]
- 禁用表达: [列举]

【开头规则】
- 字数: [X-Y字]
- 必须包含: [痛点/冲突/颠覆认知]
- 节奏: [快节奏/层层递进]
- 禁用: [直接讲大道理/冗长铺垫]

【正文规则】
- 段落长度: [每段不超过X字]
- 段落间距: [空行/紧凑]
- 案例密度: [每X段一个案例]
- 逻辑推进: [并列/递进/对比]
- 人称使用: [你/我/我们]

【语言规则】
- 句式偏好: [短句为主/长短结合]
- 语气: [亲切/犀利/温暖/专业]
- 修辞: [比喻/排比/反问]
- 禁用词: [列举]

【结尾规则】
- 字数: [X-Y字]
- 必须包含: [行动召唤/金句/互动]
- CTA: [引导在看/评论/转发]
- 情绪: [升华/共鸣/启发]
```

#### 2.2 写入 MEMORY.md

**将提炼的规则写入长期记忆：**

```markdown
# MEMORY.md 结构示例

## 核心写作风格

### 身份定位
- 我是[身份]，专注[领域]
- 读者主要是[人群特征]
- 内容定位：[干货/观点/故事]

### 固定偏好
- 标题必须用数字或冲突
- 开头不超过100字，必须有痛点
- 段落不超过3行，必须多空行
- 结尾必须有金句+CTA

### 禁用表达
- 不用"众所周知"
- 不用"不得不承认"
- 不用长句堆砌
- 不用说教语气

### 结构模板
【标题】数字+冲突+利益
【开头】痛点+冲突+钩子（100字内）
【正文1】观点+案例+金句
【正文2】观点+案例+金句
【正文3】观点+案例+金句
【结尾】金句+升华+CTA
```

#### 2.3 创建 Skill

**将方法打包成可复用的 Skill：**

```markdown
# SKILL.md 结构示例

## 技能名称
写作风格执行器

## 触发条件
用户要求写文章/创作内容

## 执行流程

### Step 1: 加载记忆
- 读取 MEMORY.md 中的核心风格
- 确认身份定位、读者画像
- 提取固定偏好和禁用表达

### Step 2: 分析需求
- 明确文章主题
- 确定目标读者
- 判断内容类型（干货/观点/故事）

### Step 3: 生成大纲
- 按结构模板设计框架
- 确定每部分核心观点
- 规划案例和金句位置

### Step 4: 撰写正文
- 严格遵循语言规则
- 控制段落长度
- 确保节奏流畅

### Step 5: 质量检查
- 对照禁用表达清单
- 确认结构完整
- 检查 CTA 是否到位

## 输出标准
- 标题：数字/冲突 + 利益点
- 开头：100字内，痛点+钩子
- 正文：每段≤3行，案例密度适中
- 结尾：金句+CTA

## 迭代记录
- [日期] 初始版本创建
- [日期] 增加禁用表达清单
- [日期] 优化结构模板
```

---

### Step 3: 生成内容（记忆 → 输出）

#### 3.1 自动加载

当 Skill 被触发时，自动完成以下动作：

```
用户: "写一篇关于[主题]的文章"

AI (内部处理):
  1. 检测到写作请求
  2. 自动加载 SKILL.md (写作风格执行器)
  3. 读取 MEMORY.md (核心风格规则)
  4. 提取: 身份定位、读者画像、固定偏好、禁用表达
  5. 执行 SKILL 中的 5 步流程
  6. 输出符合风格的文章
```

#### 3.2 质量自检

每篇文章生成后，自动对照检查：

```
质量检查清单
==============

□ 标题是否符合规则（数字/冲突/利益）
□ 开头是否在100字以内
□ 开头是否有痛点/钩子
□ 每段是否不超过3行
□ 是否多空行保持节奏
□ 案例密度是否适中
□ 是否使用了禁用表达
□ 结尾是否有金句
□ 结尾是否有CTA
□ 整体语气是否符合人设
```

### Step 4: 复盘迭代（输出 → 更新）

#### 4.1 数据复盘

基于文章发布后的数据，进行针对性优化：

```
数据复盘矩阵
============

| 数据指标 | 理想值 | 现状 | 问题诊断 | 优化方向 |
|----------|--------|------|----------|----------|
| 打开率   | >8%    | 5%   | 标题吸引力不足 | 优化标题模板 |
| 完读率   | >30%   | 15%  | 正文节奏问题 | 调整段落长度/案例密度 |
| 转发率   | >3%    | 1%   | 价值感不足 | 增加干货密度/案例质量 |
| 评论率   | >1%    | 0.5% | 互动设计弱 | 优化CTA/结尾设计 |
```

#### 4.2 规则更新

根据复盘结果，更新 MEMORY.md 和 SKILL.md：

```
迭代流程
========

1. 收集反馈
   └─ 数据表现 + 主观评价 + 读者反馈

2. 问题归因
   └─ 定位是哪个环节出问题
      ├─ 标题？→ 更新标题规则
      ├─ 开头？→ 更新开头规则
      ├─ 正文？→ 更新结构/语言规则
      └─ 结尾？→ 更新结尾规则

3. 更新记忆
   └─ 修改 MEMORY.md 中的对应规则

4. 优化流程
   └─ 更新 SKILL.md 的检查清单/流程步骤

5. 验证效果
   └─ 生成新文章 → 发布 → 收集数据 → 继续迭代
```
