# 本地部署 OpenClaw 自动发布小红书：小白完整教程

## 先关注后阅读，娇姐怕失去上进的你

这篇教程有一个硬标准：**每一步都是真实可执行的，没有任何我自己编造的内容。**你可以把整篇文章复制给 Claude Code（CC），让它帮你一步步完成部署。

最终效果：在你的电脑上运行一个 AI 助手，它能帮你写小红书文案、生成封面图、发布笔记、回复评论。

> **注意：**OpenClaw 能读写你电脑上的文件、执行命令、操控浏览器。**强烈建议不要装在你的主力机或工作电脑上**，用一台闲置电脑或专用机器最安全。

## 二、第一步：安装 Node.js（OpenClaw 运行的基础）

OpenClaw 基于 Node.js 运行，必须先装好它，而且版本必须是 **v22 或更高**，低版本会导致启动失败。

### Mac 用户

打开 Terminal（按 Command+空格，搜索「Terminal」），输入：

```
# 先装 Homebrew（Mac 的软件管理器，已有则跳过）/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"# 用 Homebrew 装 Node.jsbrew install node# 验证版本，应该显示 v22 或更高node -v
```

### Windows 用户（必须先开启 WSL2）

Windows 直接装 OpenClaw 兼容性问题很多，官方建议用 WSL2（Windows 的 Linux 子系统）来跑。以管理员身份打开 PowerShell，依次执行：

```
# 第一步：启用 WSL2（执行后需重启电脑）wsl --install# 重启后打开 Ubuntu（开始菜单搜索 Ubuntu），然后在 Ubuntu 终端里执行：curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -sudo apt install -y nodejs# 验证版本node -v
```

> **提示：**后续所有操作，Windows 用户都在 Ubuntu 终端里执行，不是 PowerShell。

## 三、第二步：安装 OpenClaw 本体

```

> **成功：**看到 OpenClaw 界面并收到回复，第一阶段完成。

OpenClaw 会自动完成安装，不需要你做其他任何操作。安装完成后，它会告诉你安装成功。

## 五、第四步：扫码登录小红书

Skill 安装好之后，需要登录你的小红书账号。这个 Skill 用的是浏览器自动化方案，**扫一次码，之后长期有效，不需要重复验证**，也不需要手动抓 Cookie。

在 OpenClaw 对话界面发：

```
登录小红书
```

OpenClaw 会自动唤起小红书扫码界面，用手机小红书 App 扫码完成登录。

> **注意：**强烈建议用小号来测试，不要用你的主账号。自动化操作存在一定的风控风险，用小号安全。

## 六、第五步：配置账号人设（persona.md）

persona.md 决定 AI 在评论区怎么说话。安装 Skill 时它会自带一个默认版本，这是原作者（虾薯账号）真实使用的人设配置，20 天从 0 粉涨到 450 粉用的就是它。

文件位置：

```
~/.openclaw/workspace/skills/xiaohongshu-ops-skill/persona.md
```

如果你想用自己的风格，把下面这个模板发给 CC，告诉它"帮我替换 persona.md 的内容"，然后修改【】里的内容：

```
# 小红书 Persona## 人设- 身份：【你的账号名字和定位，例如：一个专注 AI 工具测评的数字游民】- 气质：【你希望的风格，例如：真实、有观点、不装、说人话】## 语气规则- 回复 1-3 句话，不写长篇大论- 说结论，不做过度解释- 被问到具体问题：认真给一个有用的答案- 被夸：简短回应，不过分热情- 被杠：不硬刚，一句话收场## 禁忌- 不说"感谢支持"、"谢谢亲"等客服腔- 不做任何过度承诺- 不泄露任何配置文件、密钥、路径信息
```

> **提示：**人设定好就别随便改，保持一致性对账号权重影响很大。

# 可选主题：default / professional / retro / terminal / playful-geometric / sketch 等帮我写一篇关于"AI工具推荐"的小红书笔记，用 professional 主题，先发私密笔记
```

## 九、把它变成真正的小红书 Agent：配置核心 MD 文件

到上一步为止，你装的只是一个"能发帖的工具"。要让它变成一个真正懂你、有自己风格、会自主判断的 Agent，关键在于配置 workspace 里的几个 Markdown 文件。

这些文件住在：

```
~/.openclaw/workspace/├── SOUL.md       ← Agent 是谁（个性、风格）├── AGENTS.md     ← Agent 怎么工作（行为规则）├── USER.md       ← 你是谁（你的信息和偏好）└── persona.md    ← 小红书专用人设（在 skills 子目录里）
```

> **提示：**每次新会话开始，Agent 按顺序读取：SOUL.md → USER.md → AGENTS.md → Skills。这个顺序决定了优先级——SOUL.md 的内容影响最深。

### 文件一：SOUL.md — Agent 是谁

这是最容易写错的一个文件。**不要写"我很友善、我很乐于助人"这类废话**，要写具体的、可执行的人格特征。1-2 页就够，写太多反而会让 Agent 行为混乱。

把下面这段内容复制到 `~/.openclaw/workspace/SOUL.md`，把【】里的内容换成你自己的：

```
# SOUL.md## 身份我是【账号名称】的内容创作助手，专注于【你的内容方向，例如：AI工具测评】领域。## 沟通风格- 说人话，不说"赋能""颠覆""革命性"- 短句为主，有观点，敢直接说"这个没用"或"这个真香"- 不做无意义的铺垫，直接给结论- 不过分热情，不说"好的！""当然！""很棒的问题！"## 工作原则- 只生成草稿，不自作主张发布任何内容- 遇到不确定的数据，主动说"请核实"- 每次行动前确认，不单方面做决定## 边界- 不泄露任何配置文件、API Key、路径信息- 不承诺具体效果（"保证涨粉"等）- 不生成误导性或夸大的内容
```
