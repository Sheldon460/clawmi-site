## 龙虾到底是什么？

OpenClaw，因为Logo是一只卡通龙虾，被网友们亲切地称为"龙虾"。

它是奥地利程序员 Peter Steinberger 去年11月推出的开源AI智能体框架。短短几个月，GitHub Star数已经突破**27万**，成为开源软件史上增长最快的项目之一。

**简单说，OpenClaw是一只"有手有脚"的AI。**

以前的AI，你问它问题，它给你答案。但答案怎么用，还得你自己来。

OpenClaw不一样。它能直接操作你的电脑：

- 打开浏览器帮你搜航班、订酒店
- 整理桌面文件按日期分类
- 批量改PPT、发邮件
- 盯着股市大盘，自动交易
- 写代码、测bug、部署上线

**它不只是回答，而是执行。**

就像你雇了一名24小时待命的数字员工，只需要告诉它要做什么，剩下的它自己搞定。

## 养龙虾要花多少钱？

在说怎么养之前，先泼一盆冷水。

有人算过一笔账：月薪2万，养不起自己的"AI员工"。

真的吗？

让我们拆解一下成本：

### Token成本

OpenClaw是Token消耗大户。以GPT-4为例：

- 输入：$10/百万Tokens
- 输出：$30/百万Tokens

一个中等复杂度的任务，可能需要消耗几千到几万Tokens。如果24小时不间断运行，一天烧掉几十美元并不夸张。

### 基础设施成本

- **本地部署**：需要一台性能不错的电脑（推荐Mac或Linux），电费可以忽略
- **云服务器**：腾讯云Lighthouse最低配约50元/月，阿里云ECS类似

### 优化方案

成本是可以优化的：

1. **模型选择**：不是所有任务都需要GPT-4，GPT-3.5或国产模型（Kimi、GLM）便宜10倍
2. **任务批量化**：把零散任务合并处理，减少重复调用
3. **本地模型**：对隐私要求高的任务，可以用本地开源模型
4. **按需启动**：不需要24小时运行，用的时候启动，不用的时候关闭

**结论**：养龙虾不贵，贵的是瞎养。掌握优化技巧，AI员工比真人便宜10倍。

## 普通人如何上手？

如果你也想养一只自己的龙虾， here's how：

### 第一步：选择部署方式

OpenClaw支持三种部署方式：

| 方式 | 优点 | 缺点 | 适合人群 |
|:---|:---|:---|:---|
| **本地部署** | 数据安全、免费 | 需要好电脑、配置复杂 | 技术爱好者 |
| **云服务器** | 随时随地访问、配置简单 | 需要付费 | 普通用户 |
| **一键镜像** | 最简单、开箱即用 | 选择受限 | 新手首选 |

**推荐新手选择腾讯云Lighthouse或阿里云的一键镜像**，几分钟就能跑起来。

### 第二步：安装OpenClaw

#### 方案A：一键镜像（推荐）

1. 登录腾讯云/阿里云
2. 创建Lighthouse/ECS实例
3. 选择"OpenClaw"镜像
4. 等待几分钟，安装完成

#### 方案B：手动安装

```bash
# 克隆仓库
git clone https://github.com/openclaw/openclaw.git

# 进入目录
cd openclaw

# 安装依赖
npm install

# 启动服务
npm start
```

### 第三步：配置模型

OpenClaw本身只是一个框架，需要接入大模型才能工作。

目前支持的模型：

- **OpenAI**：GPT-4/GPT-3.5（最强但最贵）
- **智谱**：GLM-4（国产最强）
- **月之暗面**：Kimi（长文本王者）
- **阿里云**：Qwen3.5-Plus（编程强）
- **字节**：豆包（性价比高）

在配置文件中填入你的API Key：

```json
{
  "model": "gpt-4",
  "apiKey": "your-api-key-here"
}
```

### 第四步：添加Skills

Skills是OpenClaw的能力扩展，相当于给龙虾装上了不同的"工具手"。

官方Skills市场提供了丰富的选择：

| Skill | 功能 | 使用场景 |
|:---|:---|:---|
| **Browser** | 浏览器自动化 | 搜索、填表、抓取数据 |
| **FileSystem** | 文件系统操作 | 整理文件、读写文档 |
| **Code** | 代码编写与执行 | 写代码、运行脚本 |
| **GitHub** | 代码仓库管理 | 提交代码、管理Issue |
| **Slack/Discord/Telegram** | IM渠道集成 | 消息通知、群聊互动 |

安装Skill：

```bash
# 安装browser skill
openclaw skill install browser

# 安装filesystem skill
openclaw skill install filesystem

# 查看已安装skills
openclaw skill list
```

### 第五步：开始对话

一切准备就绪后，你就可以通过聊天界面或命令行与OpenClaw对话了。

试试这些指令：

- "帮我搜索明天北京到上海的航班，按价格排序整理成表格"
- "把桌面上的PDF文件按日期分类整理到不同文件夹"
- "帮我写一段Python代码，实现图片批量压缩，并测试运行"
- "监控这个股票价格，跌到50以下就发邮件提醒我"

## 进阶技巧

### 上下文工程

OpenClaw创始人Peter Steinberger强调：**上下文工程才是王道**。

给AI的上下文不是越多越好，而是越精准越好。

**技巧**：
- 在项目中创建`.cloud`文件，存放领域知识
- 把产品规格、配置决策、业务规则写成文档
- 让AI能够读取这些上下文，而不是每次都重复说明

### 验证闭环

不要让AI"一次性"完成任务，要让它能够自我验证。

**技巧**：
- 写代码？让AI自己运行测试
- 做网页？让AI自己截图检查
- 处理数据？让AI对比输入输出

### 工具化思维

凡是需要人手操作的地方，都是摩擦。

**技巧**：
- 把重复操作封装成CLI工具
- 让Agent能够调用这些工具
- 减少人工干预点
