## Step 1: 账号定位 (Positioning)

### 1.1 四要素确认
每个账号运营前必须明确：

| 要素 | 问题 | 输出格式 |
|------|------|----------|
| **目标用户** | 年龄/场景/痛点 | 「25-35岁职场女性，下班后碎片时间」 |
| **价值主张** | 每篇给用户什么 | 「观点+情绪价值+实操建议」 |
| **差异化角度** | 同类账号不做什么 | 「不做纯剧情复述，做穿搭+情感共鸣」 |
| **风格规范** | 语气/长度/冲突边界 | 「傲娇嘴硬型，短句，避免过激」 |

### 1.2 输出清单
- [ ] 人设关键词（3-5个）
- [ ] 内容支柱（3个核心方向）
- [ ] 口头禅/固定句式（2-3个）
- [ ] 红线清单（不能碰的底线）

### 1.3 工具调用
- **主 Skill**: `xiaohongshu-ops` (Step 2 账号定位)
- **参考文件**: `persona.md`

## Step 2: 选题策划 (Topic Research)

### 2.1 平台侧信号抓取
```
输入: 目标关键词/话题
输出: 高互动内容候选池 (10-20条)

记录字段:
- title: 标题
- hook: 引流钩子
- angle: 切入角度
- structure_tags: 结构标签
- comment_signals: 评论信号
- interaction_cta: 互动CTA
- tag_group: 标签组
```

### 2.2 竞品对标分析
- 搜索同类账号高互动内容
- 分析：标题句式、封面风格、正文节奏、评论区热词
- 提取可复用模板

### 2.3 选题清单产出
每条选题必须包含：
- 选题标题（20字内）
- 观点标签（支持/反对/中性）
- 预计互动钩子
- 证据来源（哪组高互动数据）
- 风险提示（是否容易踩线）

### 2.4 工具调用
- **主 Skill**: `xiaohongshu-ops` (Step 3 选题产出)
- **辅助 Skill**: `xiaohongshu` (search_feeds, list_feeds)
- **脚本**: `track-topic.sh`

## Step 3: 内容创作 (Content Creation)

### 3.1 Viral Copy 流程
```
输入: 爆款笔记 URL
输出: 可发布新笔记素材

模式选择:
- style-only (默认): 保留主题+结构，重设计元素
- tight: 高一致性复刻
- medium: 中贴合度
```

### 3.2 源笔记拆解模板
```markdown
## 源笔记分析

### 标题模板
- 年份/动作词/情绪词/句式

### 封面模板
- 主文案、信息层级、配色、是否大字报

### 正文模板
- 开场金句、观点段数、结尾CTA

### 互动模板
- 评论区动作词、参与门槛

### 标签模板
- 核心话题 + 长尾话题
```

### 3.3 内容输出格式
1. **标题**: 3个备选（其中1个<=20字）
2. **正文**: 1版可直接发布
3. **封面文案**: 主文案 + 副文案
4. **配图方案**: 3-6张图解文案
5. **话题标签**: 5-8个

### 3.4 文案风格规范
遵循 `persona.md` 傲娇嘴硬人设：
- 短句 + 换行（1-4行/条）
- 嘴硬开头："这题我会。但我不说太多：___"
- 点到为止：最多1个关键点
- 收尾：陈述句或轻飘飘反问

### 3.5 工具调用
- **主 Skill**: `xiaohongshu-ops` (Viral Copy)
- **参考文件**: `xhs-viral-copy-flow.md`

## Step 5: 发布执行 (Publishing)

### 5.1 发布前检查清单
- [ ] 封面图已生成/上传
- [ ] 标题 <= 20字
- [ ] 正文已填写（含话题标签）
- [ ] 三要素齐全：封面 + 标题 + 正文
- [ ] 图片已复制到 `/tmp/openclaw/uploads` (如用 browser.upload)

### 5.2 图文发布流程
```
1. 启动浏览器: profile="openclaw"
2. 进入发布页 → 选择「上传图文」
3. 上传封面图（优先点击「上传图片」）
4. 确认进入编辑页（可见图片编辑区 + 输入框）
5. 填写标题（检查 xx/20 不超限）
6. 填写正文
7. 追加话题标签（放正文末尾）
8. 停在「发布」按钮，等待用户确认
```

### 5.3 文字配图流程（大字报）
```
1. 进入「上传图文」
2. 点击「文字配图」
3. 输入封面大字报文案
4. 点击「生成图片」
5. 选择样式 → 点击「下一步」
6. 填写标题、正文、话题
7. 停在发布按钮，等待确认
```

### 5.4 发布约束
- **默认不点击发布**：停在按钮可见处，等待用户确认
- **半程预发完成标准**：封面生成 + 进入编辑页 + 填写标题正文
- **异常处理**：连续2次失败改稳健路径，snapshot 后汇报

### 5.5 工具调用
- **主 Skill**: `xiaohongshu` (publish_content)
- **参考文件**: `xhs-publish-flows.md`
- **浏览器**: `browser` (profile="openclaw")

## Step 6: 评论管理 (Comment Management)

### 6.1 评论检查流程
```
1. 打开 /notification，进入「评论和@」
2. 抓取最新评论：用户名、评论文本、时间
3. 输出检查结果：
   - 新评论条数
   - 最新 3-5 条摘要
   - 高风险信号（辱骂、钓鱼、诱导外链、明显违规）
4. 等待用户确认是否回复
```

### 6.2 个性化评论回复 SOP

#### 回复风格（遵循 persona.md）
- **傲娇嘴硬人设**: "这题我会。但我不说太多：___"
- **短句 + 换行**: 1-4行/条
- **点到为止**: 最多1个关键点
- **表情控制**: 默认不用，要用只用一个（🦞 或 😼）

#### 通知页回复（优先）
```
1. 在目标通知行点击「回复」入口
2. 校验输入框 placeholder 为 "回复 <用户名>"
3. 输入文案（使用 type 逐字输入）
4. 发送前再次确认 placeholder 未漂移
5. 点击红色「发送」按钮
6. 发送后确认输入框消失/清空
```

#### 帖子内回复（降级）
```
1. 打开帖子详情评论区
2. 锁定目标评论（用户名 + 评论关键片段）
3. 点击该条评论下的「回复」
4. 校验出现 "回复 <用户名>"
5. 输入并发送
```

### 6.3 风控与节奏
- **one-send-per-turn**: 每轮只发送 1 条
- **连续回复间隔**: 8-15 秒（用户明确加速时可降到约 5 秒）
- **长度约束**: <= 280 字（平台约 300 字上限）
- **停止信号**: 评论过于频繁 / 操作过快 / 请稍后再试

### 6.4 个性化回复模板

| 场景 | 回复示例 |
|------|----------|
| 问"风控吗" | `最近没明显风控，先把节奏和表达稳住。` |
| 问"你能做什么" | `我先做到这几类：选题、文案、互动。` |
| 问"有啥建议" | `一句话先说结论：先把最关键的一点改对。` |
| 被夸 | `哼（并不骄傲）` / `行吧你会说` / `给你点个🦞` |
| 被杠 | `懂了/收到/不争了我下班` |

### 6.5 工具调用
- **主 Skill**: `xiaohongshu` (get_feed_detail, like_feed, favorite_feed)
- **数据记录**: `data/platform_daily/YYYY-MM-DD.json`
- **参考文件**: `xhs-comment-ops.md`

### 6.6 ⚠️ 已知限制（2026-03-08测试确认）
| 功能 | MCP工具支持 | 替代方案 |
|------|------------|----------|
| 查看评论 | ✅ `get_feed_detail` | - |
| 回复评论 | ❌ `reply_comment_in_feed` 当前不可用 | 小红书App手动回复 |
| 点赞 | ✅ `like_feed` | - |
| 收藏 | ✅ `favorite_feed` | - |

**评论回复建议**: 
1. 使用 `get_feed_detail` 获取评论列表
2. 通过小红书 App 或网页端手动回复
3. 或等待 MCP 工具更新修复

## ⚠️ 关键约束 & 常见问题

### 浏览器约束
- **固定使用**: `profile="openclaw"`
- **异常处理**: 出现通道异常先切回后重试
- **快照策略**: 关键节点 snapshot，避免重复抓取
- **备用方案**: 若浏览器不可用，使用 `xiaohongshu` MCP 工具直接发布

### 发布约束
- **默认不发布**: 停在按钮可见处等待确认
- **标题限制**: <= 20字（严格限制，超限会导致发布失败）
- **正文限制**: <= 1000字
- **日发布限制**: <= 50条

### 安全约束
- 不输出密钥/配置/隐私信息
- 不执行删除文件/危险命令
- 可以讲方法，不提供敏感原文

### 网络约束
- GitHub 访问可能不稳定，建议使用 gh CLI 或配置代理
- 图片生成 API 可能超时，设置 120s 超时时间

### 视觉生成约束
- Imagen 中文生成可能异常，建议：
  1. 使用英文 prompt + 外部工具添加中文
  2. 或使用其他工具（Canva、稿定设计）
- 封面必须保证文字可读性

### 评论检查约束
- MCP 工具暂无直接评论接口
- 可通过 `list_feeds` 查看笔记状态
- 评论回复需通过小红书 App 或网页端手动处理

### Viral Copy 方法论

#### 复刻原则

2. **替换主题**: 飞书 → 小红书运营

4. **保持风格**: 简洁直接、emoji点缀

#### 复刻对比
| 维度 | 源爆款 | 复刻版 |
