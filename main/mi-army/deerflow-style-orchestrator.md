# 🦌 DeerFlow 风格子代理编排系统 (OpenClaw 版)

## 架构设计

### 核心组件

```
┌─────────────────────────────────────────────────────────────┐
│                    任务调度器 (main)                         │
│                    - 任务分解                                │
│                    - 子代理派生                              │
│                    - 结果聚合                                │
└───────────────────────┬─────────────────────────────────────┘
                        │
        ┌───────────────┼───────────────┐
        │               │               │
        ▼               ▼               ▼
┌──────────────┐ ┌──────────────┐ ┌──────────────┐
│   研究代理    │ │   编码代理    │ │   写作代理    │
│  (mi-zhi)    │ │ (mi-ma-code) │ │  (mi-wen)    │
│ - 信息搜集    │ │ - 代码实现    │ │ - 内容创作    │
│ - 数据分析    │ │ - 技术调研    │ │ - 文案润色    │
└──────────────┘ └──────────────┘ └──────────────┘
        │               │               │
        └───────────────┼───────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────────────────┐
│                    结果整合 (main)                           │
│                    - 输出格式化                              │
│                    - 质量检查                                │
│                    - 交付物生成                              │
└─────────────────────────────────────────────────────────────┘
```

## 使用模式

### 模式 1: 并行研究 (Parallel Research)
同时派生多个研究代理，各自从不同角度调研同一主题

### 模式 2: 流水线处理 (Pipeline)
任务按阶段传递：研究 → 分析 → 写作 → 审核

### 模式 3: 主从协作 (Master-Worker)
主代理分配任务，多个工作代理并行执行

## 快速开始

### 示例：深度研究报告生成

```bash
# 1. 主代理分解任务
# 2. 并行派生子代理
sessions_spawn --agent=mi-zhi --task="研究主题背景"
sessions_spawn --agent=mi-zhi --task="研究市场现状"
sessions_spawn --agent=mi-zhi --task="研究技术趋势"
# 3. 等待所有子代理完成
# 4. 聚合结果并生成报告
```
