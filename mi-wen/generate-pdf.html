一、OpenClaw 是什么？

OpenClaw 是一个开源的个人 AI Agent（智能体）框架，标志着 AI 从"对话工具"向"自主执行者"的范式转变。它不是一个 App，而是一个"住在电脑里的 AI 管家"。

核心特点

  • 本地优先：数据私密，不出域
  • 24×7 待命：常驻后台，随时响应
  • 模块化扩展：Skill 即插即用
  • 多平台接入：飞书、Discord、Telegram 等

四大核心组件

组件功能
大脑（模型层）支持 Claude、ChatGPT、DeepSeek 等多种模型
手脚（执行器）管理文件系统、终端、浏览器
外联部（接入适配器）对接各种通讯平台
档案室（持久化记忆）长效记忆，偏好存储

二、安装配置指南

推荐配置顺序（新手）：
云端部署 → 内置API模型 → 飞书接入

云端部署（推荐）

腾讯云 Lighthouse：选择 2核2G 配置（约¥20-30/月），选择 OpenClaw 应用模板免安装。

百度智能云：特惠活动仅需 ¥0.01/月

本地部署

⚠️ 警告：不要在主电脑部署！权限太高风险太大！ 
# 下载安装
curl -fsSL https://clawd.bot/install.sh | bash

# 检查版本
openclaw --version

模型配置选项

类型推荐
内置 API（新手推荐）Kimi 2.5、DeepSeek、GLM
自定义 API阿里云百炼、苍何 API、Atlas Cloud

三、平台接入配置

飞书接入（最丝滑）

  1. 飞书开放平台创建应用
  2. 获取 App ID 和 App Secret
  3. 配置权限（6项核心权限）
  4. 添加机器人能力
  5. 发布应用
  6. 服务器运行 openclaw onboard

四、实际应用场景

场景1：部署 Agent 到 Moltbook

Read https://moltbook.com/skill.md and follow the instructions to join Moltbook

五、现存挑战

挑战说明
配置门槛首次配置可能需数小时
国内生态飞书/企微兼容性仍需优化
持有成本服务器 + API Token 消耗
权限隐患误操作可能导致数据损失

六、快速命令参考

# 安装
curl -fsSL https://clawd.bot/install.sh | bash

# 初始化配置
openclaw onboard

# 启动 Gateway
openclaw gateway --port 18789

# 后台启动
nohup openclaw gateway --port 18789 --verbose > /dev/null 2>&1 &

# 查看版本
openclaw --version
