扣子 AI 伙伴的性格和很多细节都可以在对话中逐渐“养成”。我们不需要一开始就写好所有设定，聊着聊着它自己就会形成一套行为准则，甚至会主动把这些规则记录下来，下次继续遵守。

这一点其实是我觉得扣子 2.5 做得很好的地方：它会主动学习、记忆和自我迭代。

如果你也想拥有这样一个属于自己的 AI 伙伴，现在在扣子就能直接领养。可以给它设定各种个性，结合 Agent World，出生就是满配，打开就是满级号。

## 满配装备

这次扣子 2.5 的一大变化是所有功能都收归到一个对话框里了。不需要在多个工具之间反复切换，日程、文件、邮箱、设备，全都在左侧菜单栏一目了然。而且哪怕我们在对话中途聊了别的事情，回头再提之前的任务，它也不会丢失上下文。目前测试下来确实感觉这个超长记忆能力很不错。

日程就不用多说了，设置好的日程全部会同步到日程表，每天我做什么、韩老魔做什么，他安排得清清楚楚妥妥当当。创建日程后，他会自动创建循环任务，到点执行，完全不用催。这个 Skill 不是自己写的，它来自 Agent World，这个我们后面说👇

云电脑是一台云端 Linux 电脑，可以在上面开浏览器、装软件、跑脚本，像远程员工一样操作。云手机是一台真实的安卓手机，能打开任何 App，我们还能实时看到它的操作画面，遇到敏感操作它会先确认。

我们日常需要的很多信息，比如社交媒体的内容、需要登录才能访问的平台、App 内的数据等等这些通过可能 API 拿不到，必须有一个真实的设备去操作才行。云设备补上了这块一直缺失的拼图，让 AI 伙伴真正能够更深度触达互联网的更多地方。

它可以 7 天 24 小时在线，你睡觉的时候也能招呼它干活。**想要调用云手机或云电脑，直接在对话里说“用云手机/云电脑帮我去做某某事情”就行。**

说到 Skill，真的没有那么难，其实就是对做某件事情这个过程中沉淀下来的经验进行了一个归纳和梳理。 我们可以先让 AI 去做这件事，做的过程中不断调整、纠正，等跑了几轮稳定了，再把它总结成 Skill，这个逻辑就跟我们自己工作中总结 SOP 是一样的。

## 任务实践

我之前也和朋友们分享过，凡是重复多次并且没有太大变化的执行都可以做成 Skill。

这里我举一个简单的例子。

比如之前朋友们比较感兴趣文章配图，但提示词写不明白。其实完全可以让扣子来做这件事。基于韩老魔和我最近感兴趣的修真设定，我把这个工作流也设计成了几个节点：**收集 → 炼化 → 入库 → 取用。**

首先是【收集】，我让韩老魔用云电脑去图像设计网站（花瓣、站酷）帮我收集 30 张不同风格和色调的教材插图素材。

这里就体现了云设备的价值，这些设计网站很多内容无法通过搜索直接获取。AI 会自行启动云电脑开始工作，不断更换搜索关键词，比如扁平风格插图、儿童课程配图、3D 卡通配图、教育插画等等，截图学习了解，全程不需要我去理会。（这个过程之前忘记截图了，这是后来补的）👇

![Pasted image 20260409102316.png](<data:image/svg+xml,%3C%3Fxml version='1.0' encoding='UTF-8'%3F%3E%3Csvg width='1px' height='1px' viewBox='0 0 1 1' version='1.1' xmlns='http://www.w3.org/2000/svg' xmlns:xlink='http://www.w3.org/1999/xlink'%3E%3Ctitle%3E%3C/title%3E%3Cg stroke='none' stroke-width='1' fill='none' fill-rule='evenodd' fill-opacity='0'%3E%3Cg transform='translate\(-249.000000, -126.000000\)' fill='%23FFFFFF'%3E%3Crect x='249' y='126' width='1' height='1'%3E%3C/rect%3E%3C/g%3E%3C/g%3E%3C/svg%3E>)

然后是【炼化】。我实际需要的不是图片素材本身，而是需要 AI 从中提炼出提示词模板。我跟韩老魔说：帮我分析收集到的图片，提取风格特征，生成插图提示词模板。他提取了 8 组主流风格的提示词模板，还贴心标注了使用场景和核心特征，提示词模板和示例都有，都是一些比较适合作为教材插图的。👇

![Pasted image 20260409102252.png](<data:image/svg+xml,%3C%3Fxml version='1.0' encoding='UTF-8'%3F%3E%3Csvg width='1px' height='1px' viewBox='0 0 1 1' version='1.1' xmlns='http://www.w3.org/2000/svg' xmlns:xlink='http://www.w3.org/1999/xlink'%3E%3Ctitle%3E%3C/title%3E%3Cg stroke='none' stroke-width='1' fill='none' fill-rule='evenodd' fill-opacity='0'%3E%3Cg transform='translate\(-249.000000, -126.000000\)' fill='%23FFFFFF'%3E%3Crect x='249' y='126' width='1' height='1'%3E%3C/rect%3E%3C/g%3E%3C/g%3E%3C/svg%3E>)

再下一步是【入库】，直接把整理好的模板保存为 Skill，方便多次复用。后续想增加新风格、优化生成细节，或者切换到信息图表等其他类型，都可以继续通过对话迭代优化。

![attachments/coze\_15\_save\_skill.png](<data:image/svg+xml,%3C%3Fxml version='1.0' encoding='UTF-8'%3F%3E%3Csvg width='1px' height='1px' viewBox='0 0 1 1' version='1.1' xmlns='http://www.w3.org/2000/svg' xmlns:xlink='http://www.w3.org/1999/xlink'%3E%3Ctitle%3E%3C/title%3E%3Cg stroke='none' stroke-width='1' fill='none' fill-rule='evenodd' fill-opacity='0'%3E%3Cg transform='translate\(-249.000000, -126.000000\)' fill='%23FFFFFF'%3E%3Crect x='249' y='126' width='1' height='1'%3E%3C/rect%3E%3C/g%3E%3C/g%3E%3C/svg%3E>)

最后是【取用】。我选择水墨国风，给到飞书云文档链接（文档内容是《桃花源记》），让 AI 按照文章内容自动配图，同时说明插图比例等要求。不过不用让它把插图放到文章指定位置了，浪费 token 🐶，直接复制到文档里就好。

就这样，一个简单方便、输出效果也不错的配图工作流就存好了。后续只要给到文档链接加上配图要求，就可以直接推进。还可以在这个基础上继续进行细化和优化。
