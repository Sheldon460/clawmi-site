# MI-CE-INVEST 实战打法手册

## 核心策略

### 技能开发规范 (SOP)

1. **SKILL.md 标准结构**:
   - YAML front matter: name/description/metadata
   - 核心职能
   - 使用场景 (When to Use / When NOT to Use)
   - 核心功能详情
   - 使用方法 (命令行示例)
   - 输出格式 (示例)
   - 文件结构
   - 数据源说明
   - 注意事项
   - 与其他技能协作

2. **Python 脚本规范**:
   - 使用 argparse 处理命令行参数
   - 异常处理完善
   - 日志输出清晰
   - 配置文件与代码分离

3. **配置文件规范**:
   - JSON 格式
   - 包含注释说明
   - 提供默认值

4. **港股标的规范**:
   - 必须标注完整代码 (如 00700.HK)
   - 南向资金数据有 15 分钟延迟需注明

### 行情分析师技能矩阵

| 层级 | 技能 | 功能 | 触发条件 |
|------|------|------|----------|
| 监控层 | stock-monitor | 实时监控、异动预警 | 交易时段自动运行 |
| 分析层 | stock-analysis | 8 维评分、估值模型 | 发现异动/用户请求 |
| 研究层 | hk-ai-stock-expert | 港股 AI、南向资金 | 港股标的分析 |
| 策略层 | a-stock-monitor | 多因子选股、回测 | 量化选股需求 |

### 工作流程

```
1. 监控层发现异动 (stock-monitor)
   ↓
2. 分析层深度研究 (stock-analysis)
   ↓
3. 输出投资建议 (评分 + 目标价 + 仓位)
   ↓
4. 飞书推送预警
   ↓
5. 记录到记忆系统 (日记 + MEMORY.md)
```

### 预警条件标准

```json
{
  "price_change_pct": 5.0,    // 单日涨幅 > 5%
  "rsi_overbought": 70,       // RSI > 70 超买
  "rsi_oversold": 30,         // RSI < 30 超卖
  "volume_ratio": 3.0,        // 成交量 > 3 倍均值
  "turnover_rate": 10.0       // 换手率 > 10%
}
```

### 8 维评分体系

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 基本面 | 15% | 营收增长、利润率、ROE |
| 成长性 | 15% | 营收 CAGR、净利润 CAGR |
| 估值 | 15% | PE、PB、PEG、DCF |
| 财务健康 | 10% | 负债率、现金流、利息覆盖 |
| 行业地位 | 10% | 市场份额、竞争壁垒 |
| 管理层 | 10% | 管理层背景、股权激励 |
| 技术面 | 15% | 趋势、支撑/阻力、RSI、MACD |
| 资金面 | 10% | 主力流向、北向/南向资金 |

### 综合评级标准

```
90-100 分：⭐⭐⭐⭐⭐ 强烈推荐 (Strong Buy)
80-89 分：  ⭐⭐⭐⭐ 推荐 (Buy)
70-79 分：  ⭐⭐⭐ 持有 (Hold)
60-69 分：  ⭐⭐ 谨慎 (Underperform)
<60 分：    ⭐ 卖出 (Sell)
```
