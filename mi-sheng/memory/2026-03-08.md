## 技能一：语音合成 (Edge TTS)

### 学习成果
- 掌握 edge-tts 命令行工具使用
- 学会男声/女声语音生成
- 掌握 MP3 转 OPUS (飞书格式)

### 核心命令
```bash
# 男声
edge-tts --voice "zh-CN-YunxiNeural" --text "内容" --write-media /tmp/voice.mp3

# 女声
edge-tts --voice "zh-CN-XiaoxiaoNeural" --text "内容" --write-media /tmp/voice.mp3

# 转飞书格式
ffmpeg -y -i /tmp/voice.mp3 -c:a libopus -b:a 32k /tmp/voice.opus
```

### 常用中文语音
| 语音 | 性别 | 特点 |
|------|------|------|
| zh-CN-YunxiNeural | 男声 | 自然、年轻 |

| zh-CN-YunjianNeural | 男声 | 成熟、稳重 |
| zh-CN-XiaoyiNeural | 女声 | 活泼、亲切 |

### 发送飞书

## 技能二：AI音乐生成 (Music Generator)

### 学习成果
- 理解 Composition Plan JSON 结构
- 掌握音乐生成工作流程
- 了解质量验证(QV)标准

### Composition Plan 结构
```json
{
  "positive_global_styles": ["风格1", "风格2"],
  "negative_global_styles": ["避免的风格"],
  "sections": [
    {
      "section_name": "段落名称",
      "positive_local_styles": ["局部风格"],
      "negative_local_styles": ["避免的局部风格"],
      "duration_ms": 时长毫秒,
      "lines": ["歌词行"]
    }
  ]
}
```

### 工作流程
1. 创建 Composition Plan (JSON)
2. 配置生成参数 (时长/格式/器乐)
3. 调用 AI 音乐模型 API
4. 质量验证 (QV)
5. 输出最终音频文件

### 质量验证标准
- 时长匹配: ±0.5秒误差
- 风格一致性检查
- 音频质量检查 (无失真、噪音)
- 段落过渡自然

### Lyria - AI音乐生成
**状态**: ✅ 可用

**调用示例**:
```bash
mcporter call lyria.lyria_generate_music \
  prompt:"upbeat electronic music with driving synth bass" \
  local_path:"./audio_output" \
  file_name:"music.wav"
```

**参数说明**:
- `prompt`: 音乐描述 (必填)
- `negative_prompt`: 负面提示
- `local_path`: 本地保存路径
- `file_name`: 输出文件名
- `model_id`: 默认 lyria-002
