# 自我反思: 配图工具配置迭代 (2026-04-02)

## 问题回顾

### 重复出现的问题
1. **多次报告"画图工具不可用"** - 实际上工具已安装但未正确识别
2. **重复安装相同工具** - 没有先检查现有工具状态
3. **混淆工具类型** - 将 Canvas Design (代码绘制) 误认为 AI 生图工具
4. **API Key 管理混乱** - 多次询问用户 API Key，没有记录已提供的密钥

### 根本原因分析

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 重复报告工具不可用 | 没有建立工具状态档案 | 创建 MEMORY_TOOLS.md |
| 重复安装 | 没有先检查已安装工具 | 先检查再安装 |
| 混淆工具类型 | 没有仔细阅读 SKILL.md | 安装前阅读文档 |
| API Key 管理混乱 | 没有记录已提供的密钥 | 立即写入环境变量和记忆 |

## 改进措施

### 1. 建立工具状态档案 (已完成)
- ✅ 创建 `/workspace/mi-hua/MEMORY_TOOLS.md`
- ✅ 记录所有已配置工具的完整信息
- ✅ 包含调用方式、配置参数、使用示例

### 2. 工具识别流程 (优化)
**旧流程**:
```
用户请求 → 假设工具不存在 → 搜索安装 → 发现已安装 → 重复
```

**新流程**:
```
用户请求 → 检查 MEMORY_TOOLS.md → 
├── 工具已配置 → 直接使用
└── 工具未配置 → 检查系统 → 安装 → 记录到 MEMORY_TOOLS.md
```

### 3. API Key 管理规范 (建立)
- 收到 API Key 后立即设置环境变量
- 同时记录到长期记忆
- 验证有效性后再报告给用户

### 4. 工具分类标准 (明确)

| 类型 | 代表工具 | 特点 |
|------|----------|------|
| AI 生图 | Google GenMedia MCP, EchoFlow | 需要 API Key, 生成照片级图像 |
| 代码绘制 | Canvas Design | 无需 API, 绘制几何图形 |
| 设计辅助 | nano-banana-pro-prompts | 提供提示词, 不直接生图 |
