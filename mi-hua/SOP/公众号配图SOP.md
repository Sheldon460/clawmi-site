## 标准动作

### Step 1: 分析文章配图需求
mi-hua 分析文章结构，识别需要配图的位置：

1. **封面图**（必须高质量，900×383）
2. **正文配图**（根据内容需要，800×600）

---

### Step 2: 生成配图

#### 封面图使用 Google GenMedia（高质量）

```bash
npx -y bun google-genmedia generate \
  --prompt "配图提示词" \
  --aspect 2.35:1 \
  --size 900x383 \
  --format png \
  --output 00_cover.png
```

#### 正文配图使用 Canvas Design 或 Google GenMedia

```bash
npx -y bun canvas-design generate \
  --prompt "配图提示词" \
  --format png \
  --output 01_xxx.png
```

---

### Step 3: 保存配图

- **保存路径**: `mi-xin/03-内容工厂/3-配图成品区/`
- **命名规范**:
  - `00_cover.png`（封面图，900×383）
  - `01_xxx.png`（正文图，800×600）
  - `02_xxx.png`（正文图，800×600）

### Step 4: 更新 Markdown 配图路径

## ⚠️ 重要提醒：配图检查

**发布前必须验证配图**：

1. **内容匹配检查**：配图内容是否与当前文章主题一致
2. **文件名检查**：是否为本次生成的新文件（非旧文章复用）
3. **视觉一致性**：封面图与正文图风格是否统一

### 自动检查命令

```bash
# 检查图片修改时间（确保是新生成的）
ls -lt /path/to/配图成品区/ | head -5

# 检查图片内容（人工确认）
open /path/to/配图成品区/00_cover.png
```

## 工具优先级

| 优先级 | 工具 | 适用场景 | 状态 |
|-------|------|---------|------|
| 1 | Google GenMedia (Imagen) | 封面图、高质量配图 | ✅ 可用 |

## 质量红线（5条禁令）

1. ❌ **禁止复用旧图**：每篇文章必须生成新配图
2. ❌ **禁止相对路径**：必须使用绝对路径
3. ❌ **禁止错误尺寸**：严格遵循 900×383 和 800×600
4. ❌ **禁止非 PNG 格式**：统一使用 PNG
5. ❌ **禁止内容不匹配**：配图必须与文章主题相关
