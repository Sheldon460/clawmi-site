## 🧠 关键发现

### 1. 技能架构变化
- **旧模式**: 个人技能目录 (`~/.openclaw/skills/`)
- **新模式**: 系统级技能 (`/usr/local/lib/node_modules/openclaw/skills/`)
- **优势**: 统一管理、自动更新、无需手动维护

### 2. 核心技能映射
```
代码生成 → coding-agent (系统级)
GitHub → github + github-mcp (系统级)
Obsidian → obsidian (系统级)
1Password → 1password (系统级)
MCP → mcporter (系统级)
飞书 → feishu_* (内置工具)
跨 Agent → sessions_send/spawn (内置工具)
```

### 3. 成功案例 SOP
从记忆文档中提取的关键 SOP:
- **Coding Agent**: 必须使用 `pty: true`，否则输出损坏
- **GitHub MCP**: 需要 `GITHUB_MCP_PAT` 环境变量
- **Vercel 部署**: 需设置 Git 作者为 bot 邮箱避免权限问题
- **双重产出**: 飞书文档 + Obsidian 同步
