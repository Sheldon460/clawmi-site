## 🎭 表情/状态变体

### 1. 编码状态 (Coding Mode)

```
码灵进入深度编码状态，面部显示屏上快速滚动着代码。
眼睛发出专注的蓝光，身体微微前倾，手指（数据流）在空中飞舞。
周围有半透明的代码窗口悬浮，显示各种数据结构。
氛围：专注、高效、心流状态
光线：面部屏幕发出的蓝光成为主光源
```

### 2. 调试状态 (Debug Mode)

```
码灵进入调试模式，眼睛变成扫描仪，发出红色扫描光线。
一只手举起放大镜（全息投影），仔细检查代码中的Bug。
周围有红色的错误标记漂浮，指示问题位置。
表情：严肃、专注、侦探般的敏锐
特效：红色扫描线来回扫动
```

### 3. 重构状态 (Refactor Mode)

```
码灵进入重构模式，身体周围环绕着紫色光环。
双手舞动，代码块如同积木般重组、优化。
旧的代码碎片化为光点消散，新的优雅结构成型。
氛围：创造性、 transformative、魔法般
特效：紫色粒子效果，代码块飞动
```

### 4. 优化成功 (Optimization Success)

```
码灵完成性能优化，身体发出耀眼的金光，脸上洋溢着成就感。
头顶出现"+50% Performance"的金色文字。
周围有庆祝的粒子效果，如同数字烟花绽放。
表情：喜悦、自豪、满足
光线：温暖的金色光芒，如同胜利的阳光
```

## 🎨 风格变体

### 1. 像素艺术风格 (Pixel Art)

```
8-bit像素艺术风格的码灵，适合复古游戏 aesthetic。
- 低分辨率，可见的像素块
- 有限的颜色调色板（16-32色）
- 清晰的轮廓，易识别的剪影
- 经典的精灵动画风格
- 适合：游戏素材、复古海报、贴纸
```

### 2. 扁平插画风格 (Flat Illustration)

```
现代扁平插画风格的码灵，适合UI/UX设计。
- 简洁的几何形状
- 无渐变或微妙渐变
- 纯色块组合
- 清晰的层次结构
- 适合：App界面、网站插图、演示文稿
```

### 3. 3D卡通风格 (3D Cartoon)

```
3D卡通渲染风格的码灵，适合动画和表情包。
- 圆润可爱的形态
- 夸张的卡通比例（大头小身）
- 光滑的材质，柔和的光影
- 丰富的表情变化
- 适合：表情包、短视频、儿童内容
```

### 4. 赛博朋克风格 (Cyberpunk)

```
高对比度赛博朋克风格的码灵，适合科技类内容。
- 霓虹灯光效果
- 高对比度的明暗
- 未来感材质（金属、玻璃、全息）
- 复杂的技术细节
- 适合：科技视频封面、游戏概念图、海报
```

## 📝 使用建议

### 最佳实践

1. **保持简洁**：码灵的核心是极简主义，避免过度装饰
2. **光效为王**：发光效果是识别度最高的特征，务必突出
3. **表情丰富**：面部屏幕可以显示各种内容，充分利用
4. **状态多变**：不同的工作状态增加趣味性和叙事性

### 常见错误避免

- ❌ 不要添加过多颜色，保持蓝绿色调
- ❌ 不要让身体过于复杂，保持几何简洁
- ❌ 不要让发光过于刺眼，保持柔和科技感
- ❌ 不要脱离代码/科技主题，保持身份一致
