## 📝 经验总结

1. **服务重启后定时任务可能丢失**
   - OpenClaw Gateway 服务重启后，部分定时任务配置可能失效
   - 需要定期检查 cron 任务状态

2. **手动配置更可靠**
   - 当 CLI 命令无响应时，可以直接编辑配置文件
   - jobs.json 的结构参考现有任务即可

3. **任务 ID 重用**
   - 恢复任务时使用原任务 ID，保持历史记忆一致性
   - 便于追踪和管理

## 🔧 技术细节

### OpenClaw Cron 配置格式
- 位置：`~/.openclaw/cron/jobs.json`
- 结构：JSON 格式，包含 version 和 jobs 数组
- 时间表达式：标准 cron 格式（5 字段）
- 时区：支持时区配置

### Gateway 服务管理
- 配置文件：`~/.openclaw/openclaw.json`
- 端口：默认 18789
- 启动方式：`node /path/to/openclaw/dist/index.js gateway`
